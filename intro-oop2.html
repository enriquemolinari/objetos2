Aprendieron a programar. Programaron con el objetivo de que funcione. Lo cual es 100% razonable, que funcione es siempre el primer objetivo.

¿De qué se trata Objetos 2?

Ahora, ademas de que funcione, vamos a enfocarnos en escribir código elegante

¿Que sería Código Elegante?

¿Código Elegante?

Legible: Cualquiera puede escribir código que una computadora pueda entender. Los buenos programadores escriben código que otros programadores pueden entender (Martin Fowler).

Testeable, fácil de mantener, fácil de extender, desacoplado, cohesivo, sin código duplicado. Aplicando buenas prácticas de Programación Orientada a Objetos.

Veamos un ejemplo...

class Concurso {
  ...	
  public boolean participanteInscripto(Participante participante) {
    boolean esta = false;
    int i = 0;
    List<Inscripcion> l = this.inscriptos;
    while (!esta && i < l.size()) {
      if (l.get(i).equals(participante)) {
        esta = true;
      } else {
        i++;
      }
    }
    return esta;
  }
}

Un poco más elegante ahora...

class Concurso {
  ...	
  public boolean estaInscripto(Participante participante) {
    for (Participante p : this.inscriptos) {
      if (p.equals(participante)) {
        return true;
      }
    }
    return false;
  }
}

y un poco más elegante aún...

class Concurso {
  ...	
  public boolean estaInscripto(Participante participante) {
    boolean estaInscripto = this.inscriptos.stream()
                                    .anyMatch(p -> participante.equals(p));
    return estaInscripto;
  }
}

Sin modificar la funcionalidad, re-escribimos el código para que sea mas elegante.

En este caso, solo aplicando construcciones sintácticas modernas incorporadas en nuevas versiones de Java.
